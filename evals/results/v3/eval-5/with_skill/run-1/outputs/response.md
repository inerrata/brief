Here are the Google Responsive Search Ad (RSA) specs:

| Field | How many | Character limit (each) |
|---|---|---|
| **Headlines** | up to **15** | **30 characters** |
| **Descriptions** | up to **4** | **90 characters** |
| **Display paths** | 2 fields | **15 characters** each |

A few things worth knowing so you don't waste the slots:

- **Load all 15 headlines and all 4 descriptions if you can.** More assets give Google's machine-matching more combinations to test and optimize. Aim for at least 8-10 headlines minimum.
- **Every headline has to stand alone *and* combine.** Google mixes and matches them in any order, so don't write a set that only reads correctly in sequence.
- **Pin sparingly.** Pinning a headline/description to a fixed position locks out optimization - only do it for legally-required or compliance text.
- **Vary your angles.** Put your target keyword in at least 2 headlines, then spread the rest across benefit, proof/social proof, offer, and CTA so the asset combinations stay fresh.

Bonus: **Microsoft Ads use the exact same shape** (15 headlines x 30 chars, 4 descriptions x 90 chars), so you can write once and reuse across both.

One caveat: ad platforms occasionally change limits without notice. These are current and safe to build against, but if it's for a high-stakes launch, it's worth a 10-second check against Google's live docs before you submit.
